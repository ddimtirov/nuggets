030/**
031 * <p>Utility class providing methods for dealing with <a href="https://kotlinlang.org/docs/reference/exceptions.html#checked-exceptions">
032 * checked exceptions</a> and <a href="https://dzone.com/articles/filtering-stack-trace-hell">
033 * huge stacktraces</a>. Typically, you would want to import statically the family of {@code rethrow(...)}
034 * methods, and use the others through explicit class reference.</p>
035 *
036 * <p>All methods in this class are designed to be threadsafe.</p>
037 *
038 * <h2>String conversion</h2>
039 * <p>Every once in a while we need to format an exception without actually printing it.
040 * That is where {@link Exceptions#toString(Throwable)} comes useful.</p>
041 *
042 * <h2>About Checked Exceptions</h2>
043 * <p>The theory says that we should either handle a checked exception by performing recovery/compensating action,
044 * or let it propagate up the stack. The problem is that often there is no sensible way to handle the exception, and
045 * while propagating sounds nice, for checked exceptions it also requires the methods at the top of the stack to
046 * declaring every checked exception that the invoked method may call. In the end, more often than not, they end up
047 * throwing {@code Exception} and not bothering with the details.</p>
048 *
049 * <p>Another popular approach is to wrap the checked exception in an unchecked one and rethrow. While that saves
050 * us the need to add a new exception to the declaration of every method up the call chain, it adds another unnecessary
051 * level of "caused by" exception, pushing the real exception further down in the log output.</p>
052 *
053 * <p>With this class, we propose a third approach - take advantage from the fact that the checked/unchecked exception
054 * difference is enforced by the Javac compiler and not by the Java VM. This means that if we could throw a checked
055 * exception from a method that does not declare it, we can cast away the "checkedness". To that end we add a set of
056 * methods facilitating the following use cases:</p>
057 *
058 * <ul>
059 * <li><a href="#uc1">Use Case 1: Rethrowing without declaring a checked exception</a></li>
060 * <li><a href="#uc2">Use Case 2: Enriching exception message with local information</a></li>
061 * <li><a href="#uc3">Use Case 3: Using methods throwing checked exceptions from non-throwing lambdas</a></li>
062 * <li><a href="#uc4">Use Case 4: Transforming rethrown exceptions</a></li>
063 * <li><a href="#uc5">Use Case 5: Setting default transformation for rethrown exceptions</a></li>
064 * </ul>
065 *
066 * <h2 id="uc1">Use Case 1: Rethrowing without declaring a checked exception</h2>
067 * <p>In the most simple case we would just want to throw a checked exception as if it were unchecked.
068 * Here is example of using the {@link #rethrow(Throwable)} to achieve this:</p>
069 *
070 * <pre><code>
071 * FooBar quietlyCreateFooBar() {
072 *     try {
073 *         return new FooBar();
074 *     } catch (Exception e) {
075 *         Exceptions.rethrow(e);
076 *         return null; // compiler doesn't know this line is actually unreachable, so we need return statement
077 *     }
078 * }
079 * </code></pre>
080 *
081 * <p>As the {@code rethrow()} method returns unbound generic type, it can be used as return value, regardless
082 * of the type, transforming the snippet above to the slightly shorter:</p>
083 *
084 * <pre><code>
085 * FooBar quietlyCreateFooBar() {
086 *     try {
087 *         return new FooBar();
088 *     } catch (Exception e) {
089 *         return Exceptions.rethrow(e);
090 *     }
091 * }
092 * </code></pre>
093 *
094 *
095 * <h2 id="uc2">Use Case 2: Enriching exception message with local information</h2>
096 * <p>Often, when a lower application layer throws exception, there is some extra piece of information
097 * that could help in debugging the issue. The standard way of handling that is by wrapping the exception
098 * in another exception and rethrowing. We suggest a lighter alternative of using {@link #rethrow(Throwable, String)}
099 * to prepend a string to the text of the original exception and rethrow as unchecked.</p>
100 *
101 * <pre><code>
102 * FooBar quietlyCreateFooBar(String templateId) {
103 *     try {
104 *         return new FooBar(templates.get(templateId));
105 *     } catch (Exception e) {
106 *         String layerInfo = "TemplateId=" + templateId + ":";
107 *         return Exceptions.rethrow(e, layerInfo);
108 *     }
109 * }
110 * </code></pre>
111 *
112 * <p>Think a bit before you choose between message enrichment and wrapping. When in doubt, use enrichment.</p>
113 *
114 *
115 * <h2 id="uc3">Use Case 3: Using methods throwing checked exceptions from non-throwing lambdas</h2>
116 * <p>Many APIs designed for Java8 lambdas make use of the {@code java.util.function.*} classes.
117 * This creates a problem when the lambda implementation throws exception. Consider the following
118 * code trying to map files to their sizes.</p>
119 *
120 * <p>Consider the streams API - a method to find all files larger than specified size would typically look as follows</p>
121 *
122 * <pre><code>
123 * Collection&lt;File&gt; findLargerThan(Collection&lt;File&gt; files, long size) {
124 *     return files.stream()
125 *                 .filter(it -&gt; {
126 *                     try {
127 *                          return Files.size(f.toPath()) &gt; size;
128 *                     } catch (IOException e) {
129 *                          throw new RuntimeException(e);
130 *                     }
131 *                 })
132 *                 .collect(Collectors.toList());
133 * }
134 * </code></pre>
135 *
136 * <p>By using {@link #rethrow(Callable)} the same can be rewritten more compactly as:</p>
137 *
138 * <pre><code>
139 * Collection&lt;File&gt; findLargerThan(Collection&lt;File&gt; files, long size) {
140 *      return files.stream()
141 *                  .filter(it -&gt; rethrow(() -&gt; Files.size(it.toPath()) &gt; size))
142 *                  .collect(Collectors.toList());
143 * }
144 * </code></pre>
145 *
146 * <p id="uc3a">There is another overload of {@link #rethrow(ThrowingRunnable)} that will be automatically
147 * inferred if the supplied lambda does not throw exception. Consider the following example :</p>
148 *
149 * <pre><code>
150 * void processFile(File f) throws Exception {
151 *      do something exceptional...
152 * }
153 * File processSafely(File f) {
154 *      rethrow(() -&gt; processFile(f));
155 *      return f;
156 * }
157 * </code></pre>
158 *
159 * <p id="uc3b">Often, we want to return use or return a particular value if the lambda finished without exception.
160 * In the case above, we want to return {@code f} - this is accommodated by one final overload
161 * {@link #rethrow(ThrowingRunnable, Object)}, shaving one line and improving the readability from the previous example:</p>
162 *
163 * <pre><code>
164 * File processSafely(File f) {
165 *      return rethrow(() -&gt; processFile(f), f);
166 * }
167 * </code></pre>
168 *
169 * <h2 id="uc4">Use Case 4: Transforming rethrown exceptions</h2>
170 * <p>The {@link ExceptionTransformerBuilder} API allows us fine grained control over the exceptions we throw.
171 * Use {@link #rethrowTransformed(Throwable, boolean)} to setup an ad-hoc transformation for rethrown exception.</p>
172 *
173 * <pre><code>
174 * FooBar quietlyCreateFooBar() {
175 *     try {
176 *         return dependencyInjector.instantiate(FooBar.class);
177 *     } catch (Exception e) {
178 *         return rethrowTransformed(e, true) // true means we want to ignore any default transforms (rather than adding to them)
179 *                .replaceMessage(t -&gt; t.getMessage() + " while creating foobar") // add extra info
180 *                .unwrapThese(InvocationTargetException, ExecutionException)     // remove unnecessary levels of indirection
181 *                .done();
182 *     }
183 * }
184 * </code></pre>
185 *
186 * <h2 id="uc5">Use Case 5: Setting default transformation for rethrown exceptions</h2>
187 * <p>While {@code rethrowTransformed()} gives us a lot of flexibility, sometimes we just want to set and forget
188 * an exception cleaning transformation for the whole application. That is what {@link #transformOnRethrow()} is
189 * about. If you configure a transformer with this method it will be applied automatically on every exception
190 * rethrown on the current thread and its children. That is, unless the application was started with {@code <b>-D</b>nuggets.Exceptions.transform-on-rethrow=false}
191 * command line parameter.</p>
192 *
193 * <p>You would usually want to call this method in the very beginning from your main method (which makes sure it would
194 * apply for all application threads). The method can be called any time and the transformation applies for the current
195 * thread and any future child threads.</p>
196 *
197 * <pre><code>
198 * Exceptions.transformOnRethrow()
199 *           .replaceStackTrace(ExceptionTransformerBuilder.FILTER_GROOVY_SYNTHETIC)
200 *           .filterStackFramesForClass(Pattern.compile("^(java.lang.reflect|sun.reflect).*"))
201 *           .filterStackFramesForClass(Pattern.compile("^(org.codehaus.groovy|org.gradle).*"))
202 *           .unwrapThese(InvocationTargetException.class)
203 *           .unwrapWhen(ex -&gt; ex.getMessage() == null &amp;&amp; ex.getClass().equals(RuntimeException.class))
204 *           .done();
205 * </code></pre>
206 *
207 * <p id="uc5a">The library even has a shortcut for the above, that you can apply by calling {@link #setupTransformOnRethrowDefaultConfig()}</p>
208 */
209public class Exceptions {
210    private static final boolean TRANSFORM_ON_RETHROW = Boolean.parseBoolean(System.getProperty("nuggets.Exceptions.transform-on-rethrow", "true"));
211    private static final InheritableThreadLocal<Function<Throwable, Throwable>> TRANSFORMER = new InheritableThreadLocal<>();
212    private Exceptions() {}
213
214    /**
215     * <p>Configures a transformer that would be used for every rethrown exception,
216     * unless a System property {@code nuggets.Exceptions.transform-on-rethrow}
217     * is not set explicitly to {@code false}</p>
218     *
219     * <p>This method sets up reasonable defaults that may change form version to
220     * version of this library. If you need more control, use {@link #transformOnRethrow()}
221     * instead.</p>
222     *
223     * @see #transformOnRethrow()
224     * @see <a href="#uc5a">Use Case 5 for example usage</a>
225     */
226    public static void setupTransformOnRethrowDefaultConfig() {
227        transformOnRethrow()
228                .filterStackFramesForClass(Pattern.compile("^(java.lang.reflect|sun.reflect).*"))
229                .filterStackFramesForClass(Pattern.compile("^(org.codehaus.groovy|org.gradle).*"))
230                .unwrapThese(InvocationTargetException.class)
231                .unwrapWhen(ex -> ex.getMessage() == null && ex.getClass().equals(RuntimeException.class))
232                .build();
233    }
234
235    /**
236     * <p>Configures a transformer that would be used for every rethrown exception,
237     * unless a System property {@code nuggets.Exceptions.transform-on-rethrow}
238     * is not set explicitly to {@code false}</p>
239     *
240     * <p>The transformer is thread-local and inherited from children threads.
241     * This means that you'd better set it early in the application main thread.
242     * It is acceptable to redefine the transformer multiple times (i.e. worker
243     * threads wanting to clean up the thread-pool stack-frames)</p>
244     *
245     * <p>Keep in mind that this transformer is not automatically applied to the
246     * argument of {@link #toString(Throwable)}, so one can easily dump the full
247     * stack if needed.</p>
248     *
249     * <p>If you are not sure what transformations to specify, you may want to just
250     * call {@link #setupTransformOnRethrowDefaultConfig()} at application startup,
251     * which would configure reasonable defaults.</p>
252     *
253     * @return a builder to configure the cleanup. Call {@link ExceptionTransformerBuilder#done()} to install.
254     *
255     * @see #setupTransformOnRethrowDefaultConfig()
256     * @see <a href="#uc5">Use Case 5 for example usage</a>
257     */
258    public static @NotNull ExceptionTransformerBuilder transformOnRethrow() {
259        return new ExceptionTransformerBuilder(TRANSFORMER::set);
260    }
261
262    /**
263     * <p>Rethrows exception, but not before cleaning it up with ad-hoc specified transformation.
264     * The transformation will be applied <em>before</em> OR <em>instead-of</em> the
265     * the default transformation, depending on the value of the {@code suppressDefaultTransform}
266     * parameter.</p>
267     *
268     * @param t an exception to format and rethrow.
269     *
270     * @param suppressDefaultTransform if {@code false} the default transformation, specified
271     *                                 by {@link #transformOnRethrow()} will be applied AFTER the
272     *                                 transform specified by this method. Otherwise, ONLY the
273     *                                 this transform will be applied (the default will be suppressed).
274     *
275     * @return a builder that would rethrow the formatted {@code t} when the
276     *        {@link ExceptionTransformerBuilder#done()} method is called.
277     *
278     * @see ExceptionTransformerBuilder
279     * @see #transformOnRethrow()
280     * @see #rethrow(Throwable, String)
281     * @see #rethrow(Throwable)
282     * @see <a href="#uc4">Use Case 4 for example usage</a>
283     */
284    public static @NotNull ExceptionTransformerBuilder rethrowTransformed(@NotNull Throwable t, boolean suppressDefaultTransform) {
285        return suppressDefaultTransform
286                ? new ExceptionTransformerBuilder(transform -> doSneakyThrow(transform.apply(t)))
287                : new ExceptionTransformerBuilder(transform -> rethrow(transform.apply(t)));
288    }

290    /**
291     * <p>Adapter allowing to use throwing methods from void lambdas of non-throwing interfaces.
292     * Unlike {@link #rethrow(ThrowingRunnable)} this overload returns a value and can be used
293     * in {@code return} statement. For usage examples, see {@link #rethrow(Callable)}.</p>
294     *
295     * @param throwingRunnable typically a parameterless lambda inside a capturing lambda of the desired type
296     * @param returnValueOnSuccess value to return if no exception is thrown.
297     *
298     * @param <T> inferring the return type.
299     * @return the {@code returnValueOnSuccess} if the {@code throwingRunnable} completed without throwing exception.
300     * @throws Throwable thrown from the {@code throwingRunnable} (undeclared)
301     *
302     * @see #rethrow(Callable)
303     * @see #rethrow(ThrowingRunnable)
304     * @see <a href="#uc3b">Use Case 3 for example usage</a>
305     */
306    @SuppressWarnings("JavaDoc") // declaring we throw an exception not in the signature
307    @Contract(pure=true)
308    public static <T> @Nullable T rethrow(@NotNull ThrowingRunnable throwingRunnable, @Nullable T returnValueOnSuccess) {
309        try {
310            throwingRunnable.run();
311            return returnValueOnSuccess;
312        } catch (Throwable e) {
313            return rethrow(e);
314        }
315    }
316
317    /**
318     * <p>Adapter allowing to use throwing methods from void lambdas of non-throwing interfaces.
319     * For usage examples, see {@link #rethrow(Callable)}.</p>
320     *
321     * @param throwingRunnable typically a parameterless lambda inside a capturing lambda of the desired type
322     * @throws Throwable thrown from the {@code throwingSupplier} (undeclared)
323     *
324     * @see #rethrow(Callable)
325     * @see #rethrow(ThrowingRunnable, Object)
326     * @see <a href="#uc3a">Use Case 3 for example usage</a>
327     */
328    @SuppressWarnings("JavaDoc") // declaring we throw an exception not in the signature
329    public static void rethrow(@NotNull ThrowingRunnable throwingRunnable) {
330        rethrow(throwingRunnable, null);
331    }
332
333    /**
334     * <p>Adapter allowing to use throwing methods from lambdas of non-throwing interfaces.</p>
335     *
336     * @param throwingSupplier typically a parameterless lambda inside a capturing lambda of the desired type
337     * @param <T> inferred return type
338     * @return the return value of the {@code throwingSupplier}
339     * @throws Throwable thrown from the {@code throwingSupplier} (undeclared)
340     *
341     * @see #rethrow(ThrowingRunnable)
342     * @see <a href="#uc3">Use Case 3 for example usage</a>
343     */
344    @SuppressWarnings("JavaDoc") // declaring we throw an exception not in the signature
345    public static <T> T rethrow(@NotNull Callable<T> throwingSupplier) {
346        try {
347            return throwingSupplier.call();
348        } catch (Exception e) {
349            return rethrow(e);
350        }
351    }

353    /**
354     * <p>Rethrows a checked exception as unchecked, even if it is not added to the method
355     * signature. Also allows to prepend some text to the exception message (useful when we
356     * want to capture some context without wrapping the exception).</p>
357     *
358     * <p>For more examples on the usage, see {@link #rethrow(Throwable) }</p>
359     *
360     * @param t an exception to rethrow as unchecked.
361     * @param prependedReason text to prepend to the exception's message
362     * @return dummy value, allowing this method to be used as a value of return statement.
363     * @param <T> return type - inferred by the compiler, depending on the context expectations.
364     * @throws Throwable thrown from the {@code throwingSupplier} (undeclared)
365     *
366     * @see Throwable#getMessage()
367     * @see #rethrow(Throwable)
368     * @see <a href="#uc2">Use Case 2 for example usage</a>
369     */
370    @Contract("_,_->fail") @SuppressWarnings("JavaDoc") // declaring we throw an exception not in the signature
371    public static <T> T rethrow(@NotNull Throwable t, @Nullable String prependedReason) {
372        if (prependedReason!=null && !prependedReason.trim().isEmpty()) {
373            Extractors.pokeField(t, Throwable.class, "detailMessage", prependedReason + t.getMessage());
374        }
375        return rethrow(t);
376    }
377
378    /**
379     * <p>Rethrows a checked exception as unchecked, even if it is not added to the method
380     * signature. Used when we don't want to recover from an exception and propagate it
381     * straight to the top-level exception handler.</p>
382     *
383     * <p>The return type is inferred, allowing us to use this method as return expression
384     * regardless of the return type and take advantage of the compiler reachability analysis.</p>
385     *
386     * @param t an exception to rethrow as unchecked.
387     * @return dummy value, allowing this method to be used as a value of return statement.
388     * @param <T> return type - inferred by the compiler, depending on the context expectations.
389     * @throws Throwable thrown from the {@code throwingSupplier} (undeclared)
390     *
391     * @see #rethrow(Throwable, String)
392     * @see <a href="#uc1">Use Case 1 for example usage</a>
393     */
394    @Contract("_->fail") @SuppressWarnings("JavaDoc") // declaring we throw an exception not in the signature
395    public static <T> T rethrow(@NotNull Throwable t) {
396        if (TRANSFORM_ON_RETHROW) {
397            Function<Throwable, Throwable> transformer = TRANSFORMER.get();
398            if (transformer!=null) {
399                t = transformer.apply(t);
400            }
401        }
402        //noinspection RedundantTypeArguments
403        return Exceptions.<RuntimeException, T>doSneakyThrow(t);
404    }
405
406    /**
407     * Uses a mix of type inference and generics to cast away the "checkedness" of exception.
408     *
409     * @param t an exception to rethrow as unchecked.
410     * @param <T> reinterpret-casting the desired throwing type.
411     * @param <R> inferring the return type.
412     * @throws T a type system loophole, using generic cast at the caller to make javac treat the exception
413     *           as {@code RuntimeException}.
414     *
415     * @see <a href="http://stackoverflow.com/questions/31316581/a-peculiar-feature-of-exception-type-inference-in-java-8">
416     *     Sneaky throw using Java 8 type inference</a>
417     */
418    @Contract("_->fail") @SuppressWarnings("unchecked")
419    private static <T extends Throwable, R> R doSneakyThrow(@NotNull Throwable t) throws T {
420        throw (T) t;
421    }
422
423    /**
424     * Conveniently formats the full stacktrace as a String.
425     *
426     * @param t the exception to format.
427     * @return a string representing the output of {@link Throwable#printStackTrace()}
428     */
429    public static @NotNull String toString(@NotNull Throwable t) {
430        StringWriter sw = new StringWriter();
431        try(PrintWriter pw = new PrintWriter(sw)) {
432            t.printStackTrace(pw);
433        }
434        return sw.getBuffer().toString();
435    }
